# 港式职场中英夹杂听力学习网站 — MVP SPEC v1

---

## 1. Problem Statement（问题陈述）

**目标用户**：会粤语、但英语一般的香港职场人。

他们在日常职场中经常听到或需要使用英文词，例如：
`meeting`、`follow up`、`deadline`、`approval`、`client`、`handover`、`urgent`、`EOD`

**核心痛点**：
- 不是完全不会英语，而是缺少贴近粤语语境、贴近香港 office 场景、可听可模仿的学习方式
- 传统英语学习太教材化，和真实职场语境脱节
- 普通中英夹杂内容偏段子，学习价值不够

**要解决的问题**：
让会粤语但英语一般的职场人，通过短音频卡片，学习真实、自然、港式职场语境下的中英夹杂表达。

---

## 2. Proposed Solution（方案描述）

做一个**港式职场中英夹杂听力学习网站**。

第一版定位：不是课程平台、不是 AI 生成器、不是社区，而是一个**轻量 MVP**。

### 核心体验

用户打开网站后：
1. 看到一张**今日学习卡片**
2. 点击播放一段**香港职场粤语口音 + 英文词夹杂**的音频
3. 查看粤语口语句子
4. 查看主关键词和辅助关键词解释
5. 展开关键词解释
6. 下方浏览 30 条精品内容
7. 按 6 个职场场景筛选
8. 每条卡片可以反馈：**有用 / 一般 / 不地道**

---

## 3. MVP 内容范围

第一版固定制作 **30 条精品内容**，不做每日自动生成。

### 内容主题

整体风格：**香港 Office 生存场景**。拆成 6 类，每类 5 条：

| # | 场景 | 主题 |
|---|------|------|
| 1 | Meeting | 开会表达 |
| 2 | Follow up | 催进度 |
| 3 | Approval / Manager | 老板审批 |
| 4 | Client | 客户沟通 |
| 5 | Handover / Teamwork | 同事协作 |
| 6 | OT / Urgent / Deadline | 加班救火 |

### 每条内容结构

每条卡片包含：

- 粤语口语句子
- AI 粤语音频
- 主关键词 ×1
- 辅助关键词 ×1–2
- 简体中文解释
- 场景标签
- 轻反馈按钮

### 数据模型

```json
{
  "id": "001",
  "scene": "Meeting",
  "cantoneseText": "你一阵 meeting 前，记得 update 返个 deck。",
  "audioUrl": "/audio/001.mp3",
  "mainKeyword": {
    "word": "update",
    "meaning": "更新"
  },
  "supportKeywords": [
    { "word": "meeting", "meaning": "会议" },
    { "word": "deck", "meaning": "演示文稿 / PPT" }
  ],
  "explanation": "提醒对方在开会前更新一下演示文稿。"
}
```

## 4. Audio Strategy（音频策略）

### 第一版方案

使用 **AI 粤语 TTS** 生成音频，但**必须人工审核后上线**。

### 音频质量标准

每条音频上线前必须检查：

- 是否接近香港职场粤语口音
- 英文词是否清楚
- 中英切换是否自然
- 停顿是否像真实 office 对话
- 是否有明显机器感
- 是否会误导用户学习错误发音

### 不合格处理方式

1. 重新生成音频
2. 调整句子结构
3. 替换英文词
4. 必要时放弃该条内容

**核心原则：这个产品的命门不是页面，是声音可信度。**

## 6. User Interaction（用户交互）

### 必做

- 播放音频
- 重播音频
- 展开关键词解释
- 切换场景分类
- 点击轻反馈（有用 / 一般 / 不地道）
- 点击详细反馈入口

### 不做

- 登录
- 收藏
- 跟读录音
- 发音评分
- 学习进度
- 用户中心

---

## 7. Visual Direction（视觉风格）

**温暖学习卡片风 + 轻微香港 Office 元素**

### 风格关键词

- 米白、浅黄、浅橙为主色调
- 圆角卡片
- 深灰文字
- 温暖但克制
- 学习感强于娱乐感

### 可加入的轻量元素

- 便利贴
- 工牌
- 文件夹
- 会议日程
- 小型音频波形
- Office 桌面感

### 避免

不要做太重的霓虹港风，那会抢走学习重点。

## 8. Technical Constraints（技术约束）

### 技术栈

| 层 | 选型 |
|----|------|
| 框架 | Next.js |
| 内容管理 | 静态 JSON |
| 音频 | 静态文件 |
| 部署 | Vercel |
| 轻反馈存储 | Supabase |
| 基础统计 | Vercel Analytics |
| 行为事件 | 自定义事件 |

### 内容管理

第一版使用 JSON 文件管理内容。暂不使用：CMS、数据库管理内容、后台管理系统。

> 当内容超过 100 条后，再考虑迁移到 CMS 或数据库。

### 数据统计事件

- `audio_play` — 点击播放音频
- `audio_complete` — 音频播放完成
- `keyword_expand` — 展开关键词解释
- `feedback_click` — 点击有用 / 一般 / 不地道
- `scene_filter_click` — 切换场景分类

### 反馈存储

| 反馈类型 | 存储方式 |
|----------|----------|
| 轻反馈 | 存入 Supabase |
| 详细反馈 | 跳转外部表单（飞书表单 / Google Form） |

## 9. Non-goals（明确不做）

第一版明确不做：

1. 登录系统
2. 收藏功能
3. 学习进度
4. 跟读录音
5. 发音评分
6. AI 自动生成内容
7. 用户输入生成句子
8. 后台 CMS
9. 课程体系
10. 会员付费
11. 社区互动
12. 复杂数据分析后台
13. 多语言版本
14. 普通话版本
15. 真人录音版本

**第一版只验证**：用户是否愿意听、是否觉得地道、是否觉得有学习价值。

---

## 10. Success Criteria（成功标准）

上线后 7 天内重点看以下指标：

### 核心指标

| # | 指标 | 目标 |
|---|------|------|
| 1 | 音频播放完成率 | ≥ 60% |
| 2 | 单次访问平均学习 | ≥ 3 条内容 |
| 3 | "有用 / 地道"类反馈占比 | ≥ 70% |
| 4 | 收集真实用户反馈 | ≥ 20 个 |
| 5 | 是否有用户主动问"有没有更多内容？" | 有 |

### 不作为主要成功标准

- 访问量
- 页面好不好看
- 功能多不多
- 技术栈是否高级
- 是否像完整学习平台

> 访问量会骗你。真正有价值的是：用户听完、继续听、觉得有用、愿意要更多。

---

## 11. MVP 判断结论

这个项目第一版的正确方向是：

**做一个极轻、极聚焦、内容质量优先的港式职场听力卡片网站。**

不要一开始做成 SaaS。不要一开始做成课程平台。不要一开始做成 AI 生成器。不要一开始做成社区。

你现在最该打磨的不是功能，而是这三件事：

1. **30 条内容**是否真的像香港 office 会说的话
2. **AI 粤语音频**是否足够自然可信
3. 用户听完后是否觉得**"我真的学到一句能用的表达"**

只要这三个成立，后面才有资格谈扩展。
